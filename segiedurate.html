SEGi EduRate - Lecturer Evaluation System

Project Overview

Visit SEGi EduRate

System Name: SEGi EduRate

Aims: To provide an efficient platform for evaluating lecturers and managing the lecturer evaluation process.

Technologies Used: CodeIgniter, Bootstrap, PHP, JavaScript, MySQL (managed via TablePlus)

Features: -

  • Enabled admins to manage users (students & lecturers), courses, semesters, and evaluation forms.
  • Ensured only authorized students can submit evaluations anonymously, with only one submission per evaluation.
  • Automated email reminders sent only to students who have not yet filled in (automatically sent three days before the deadline).
  • Restricted lecturers to viewing reports only for their own courses.

(1) Admin Dashboard
The admin dashboard offers an overview of system activities. It displays the active semesters and evaluation progress, along with several charts highlighting key metrics. All charts can be exported in SVG, PNG, or CSV format for further analysis and reporting. The final component is a calendar, allowing users to view the semester and evaluation durations.
(2) Manage Users (Add, Modify, Delete, Add via Excel Upload)
Admins can manage users (students and lecturers) by adding, editing, deleting, or uploading users in bulk via Excel using the "Importing Students" button. The table allows for easy searching by typing or filtering by selecting the "Click to Filter" button. Additionally, the table data can be exported using the available buttons, with options to copy, save as CSV, Excel, PDF, or print.
(3) Manage Semesters
This feature allows admins to create, edit, and delete semesters. It coordinates courses, lecturers, and students by linking them to specific semesters. Admins can assign courses (from existing course data) to lecturers (selected from created lecturer data) and enroll students (from the created student list) into the appropriate courses for the selected semester.
(4) Manage Evaluations
The evaluation process is managed through two main menus: "Evaluation Questions" and "Evaluation Forms." Admins can add or import evaluation questions via Excel. Once the questions are set, they can visit the "Evaluation Forms" menu to create an evaluation form by selecting an active semester. The admin can also specify the evaluation start and end dates and choose whether to send email notifications to students and lecturers.
(5) Anonymous Evaluation Submission
To fill out the evaluation, students can go to the "Evaluation Form" menu, where a table lists only the courses they're enrolled in. Each course will have an "Evaluate" button. After submitting the evaluation, the button changes to "View." Students can then review their responses, but cannot modify them, as only one submission is allowed.
(6) Evaluation Report
Admins can view evaluation reports based on the course in a specific semester or by lecturer. On the other hand, lecturers can only view evaluation reports for the courses they have taught. The system demo starts by showing the admin view, and at 2:30, it switches to the lecturer's view of the evaluation reports.
(7) Automated Email Reminders to Students
The final highlight is the automated email reminder feature! Three days before the evaluation deadline, the system automatically sends daily reminders to students who haven't completed their evaluations. This eliminates the need for manual checks and significantly reduces the admin's workload, as the process runs entirely without human intervention.

Future Improvements

1. Mobile Responsiveness

The current design is optimized for larger screens. Future enhancements will improve responsiveness for better mobile usability.

2. Faculty-Level Hierarchy

Currently, evaluations are based on specific programs. A planned improvement is introducing a faculty-level hierarchy, grouping programs under faculties, allowing more insightful evaluations at both levels.

3. Enhanced Role-Based Access Control

Admins currently have broad privileges. A refined role-based control will be introduced:

  • Faculty Leaders: Manage programs, assign admins, and oversee enrollments.
  • Sub-Administrators: Handle semester management within their assigned faculty/program.
  • Ensures secure, structured administration aligned with institutional needs.
